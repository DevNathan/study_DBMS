## 1. DBMS 개론
### DB(Database)
    데이터가 모여 있는 기지,
    데이터만을 다루고 저장하는 곳이다.

### DBMS(Database Management System)
    DB를 관리할 수 있는 구체적인 시스템
    대표적으로 Oracle, MySQL, MS-SQL, PostgreSQL 등이 있으며
    RDBMS와 NoSQL로 더욱 상세히 나눌 수 있다(NoSQL의 경우는 여기서 다루지 않음).

### RDBMS(Related Databse Management System)
관계형 데이트베이스 시스템이라 불리며 테이블끼리 서로 관계를 맺는것이 특징이다.<br>
해당 글은
<br><br>
예를 들어,<br>
TABLE A
번호(PK) | 이름 | 나이 | 아이디(UQ)
--- | --- | --- | ---
1 | John Doe | 20 | 000123
<br>

TABLE B
주문번호(PK) | 번호(FK) | 날짜 | 상품수량
--- | --- | --- | ---
20000000 | 1 | 0101 | 5

위의 두 테이블이 "번호"를 기준으로 관계될 수 있으며 이것이 관계형 데이터베이스의 특징이다라고 말할 수 있다.
<br><br>
- Primary Key(PK)

      고유한 값.
      각 행의 구분점으로 사용된다.
      중복이 없고 NULL값을 허용하지 않는다.
- Foreign Key(FK)

		다른 테이블의 PK를 의미한다.
		보통 테이블끼리 관계를 맺을 때 사용한다.
		중복이 가능하고 NULL도 허용한다.
- Unique Key

      NULL은 허용하지만 중복은 허용하지 않는다.

### SQL(Structured Query Language)
	데이터베이스와 소통하는 언어이며 스크립트 언어로 작동한다.
#### 컴파일 언어와 인터프리터(스크립트) 언어의 차이점
- 컴파일
	- 파일 단위로 해석한다(일괄처리).
 	- 수정이 거의 없을 때 효율적이다.
- 인터프리트
	- 한 줄 단위로 해석한다(개별처리).
 	-  빈번한 수정 시 또는 부분 실행 시 효율적이다.
 
## 2. SQL문(Query 문) - DDL, DML, DCL, TCL
### 2-1 DDL
1. DDL(Data Definition Language) : 데이터 정의어
	- 테이블 조작, 제어 관련 쿼리 문

			1) CREATTE : 테이블 생성
			2) DROP : 테이블 삭제
			3) ALTER : 테이블 변경
			4) TRUNCATE : 테이블 안에 있는 내용 전체 삭제

2. 자료형(Type) : 용량은 부족할 일 없이 충분히 확보한다.
	1) 숫자

			NUMBER(precision): 정수
			NUMBER(precision, 소수점 자리수): 실수
	2) 문자열
  
			CHAR(n) : 고정형
			CHAR(4)에 'A'를 넣으면 A^^^ 빈 자리가 공백으로 채워진다.
			형식을 정한 날짜, 주민등록번호, 상태값 등 글자 수가 절대 변하지 않는 값을 넣는다.
			VARCHAR(precision), VARCHAR2(precision) : 가변형
			값의 길이만큼 공간이 배정된다. 글자 수에 변화가 있는 값을 넣는다.
			(VARCHAR2만 사용할 것)
	3) 날짜

  			DATE : FORMAT에 맞춰서 날짜를 저장하는 타입

#### 무결성
	데이터의 정확성, 일관성, 유효성이 유지되는 것

  	정확성: 데이터는 애매하지 않아야 한다.
	일관성: 각 사용자가 일관된 데이터를 볼 수 있도록 해야한다.
	유효성: 데이터가 실제 존재하는 데이터여야 한다.

1. 개체 무결성

		모든 테이블이 PK로 선택된 컬럼을 가져야 한다.
2. 참조 무결성

		두 테이블의 데이터가 항상 일관된 값을 가지도록 유지하는 것.
3. 도메인(컬럼) 무결성

		컬럼의 타입, NULL값의 허용 등에 대한 사항을 정의하고
		올바른 데이터가 입력되었는지를 확인하는 것.

#### 데이터베이스 모델링(기획)
	추상적인 주제를 DB에 맞게 설계하는 것
1. 요구사항을 분석한다

		회원, 주문, 상품 : 3가지를 관리하고자 한다.
2. 개념적 설계(개념 모델링)을 하여 각 테이블에 무엇이 필요할지 정리한다.

		TBL_회원 : 번호PK, 비밀번호, 이름, 주소, 생일, 이메일UK
		TBL_상품 : 상품번호PK, 상품명, 가격, 재고량
		TBL_주문 : 주문번호PK, 회원번호FK, 상품번호FK, 주문날짜, 주문갯수

### DB 정규화
	삽입/수정/삭제의 이상현상(Anomaly)을 제거하기 위한 작업으로, 
 	데이터의 중복을 최소하 하는데에 목적이 있다.
  	5차 정규화까지 있으나 대부분 3차 정규화까지만 한다.

#### 1차 정규화
	같은 성격과 내용의 컬럼이 연속적으로 나타날 경우 1차 정규화로 해결할 수 있다.

회원번호 | 회원이름 | 프로그램
--- | --- | --- |
1 | 홍길동 | C#, JAVA, PYTHON

<br><br>
위와 같은 테이블을 사용할 시 문제점이 문제점이 발생한다.
1. 특정 프로그램을 찾기 어려워짐 + 성능저하 동반
2. 정보 수정이 어려워짐
<br>

회원번호 | 회원이름 | 프로그램
--- | --- | --- |
1 | 홍길동 | C#
1 | 홍길동 | JAVA
1 | 홍길동 | PYTHON

따라서 아래와 같이 수정하는 것이 성능향상과 관리에 도움이 된다.<br>
한 칸에는 하나의 데이터만 보관하는 것이다.

#### 2차 정규화
	조합키(복합키)로 구성되었을 경우 조합키의 일부분에만 종속되어 속성이 있을 경우(부분 종속)

현재 테이블과 관련 없는 내용의 컬럼이 동반하여 존재할 경우 나중에 정보를 관리하는데 있어 불편을 겪을 가능성이 매우 높다. 따라서 테이블의 주제에 따라서 정보를 분리하는 것을 2차 정규화라 한다.
<br><br>
아래는 2차 정규형을 만족하는 테이블의 예시이다.
<br><br>
TBL_MEMBER

회원번호 | 회원이름 | 프로그램
--- | --- | --- |
1 | 홍길동 | C#
1 | 홍길동 | JAVA
1 | 홍길동 | PYTHON

TBL_PROGRAM

프로그램 | 강좌 기간 | 가격
--- | --- | ---

JAVA | 70일 | 220,000
PYTHON | 80일 | 300,000

#### 3차 정규화
	PK가 아닌 컬럼이 다른 컬럼을 결정하는 경우 이행함수 종속 제거

위의 예시는 사실 3차 정규형까지 만족한다고 할 수 있다. 회원은 프로그램의 정보를 토대로 자신이 지불해야할 가격까지 조회 할 수 있도록 되어있다. 즉 3차 정규형을 만족한다고 할 수 있다.
	
#### 이상현상(Anomaly)의 종류
	- 삽입 이상 : 새 데이터를 삽입하기 위해 불필요한 데이터도 삽입해야하는 문제
	- 갱신 이상 : 중복 행 중에서 일붐만 변경하여 데이터가 불일치하게 되는 모순의 문제
	- 삭제 이상 : 행을 삭제하면 꼭 필요한 데이터까지 함께 삭제되는 문제

 	이러한 일이 없게 하기 위해선 RDBMS에서는 정규화에 대해 잘 이해하고 테이블간의 관계도를 적절히 작성하는 것이 필수다.

### 2-2 DML
	DML(Data Manipulation Language) : 데이터 조작어

1. SELECT : 조회(검색)

		SELECT [컬럼명1, 컬럼명2, …]
		FROM [테이블명]
		WHERE [조건식]
				
2. INSERT : 추가

		1) 컬럼명을 생략 할 수 있으며, 이 경우 DEFAULT 제약 조건이 발생된다.
			INSERT INTO [테이블명]
			(컬럼명1, 컬럼명2, …)
			VALUES(값1, 값2, …)
		2) 모든 값을 전부 작성해야 되며, 컬럼명은 직접 작성하지 않는다.
			INSERT INTO [테이블명]
			VALUES(값1, 값2, …)
				
3. UPDATE : 수정

		UPDATE [테이블명]
		SET [기존 컬럼명1] = [새로운 값1], ...
		WHERE [조건식]
				
4. DELETE : 삭제

		DELETE FROM [테이블명]
		WHERE [조건식]

#### JOIN
	정규화를 마친 테이블 들은 결과적으로 모든 정보가 흩어져 있다.
 	이 중에서 사용자가 필요한 정보만 가져와서 가상의 테이블처럼 만들고 결과를 보여주기 위해 사용한다.
  	(조회의 성능을 향상시키기 위해 JOIN을 통해 테이블을 합친 후 사용한다.)

1. 내부 조인(INNER JOIN)

		조건이 일치하는 값만 합쳐서 조회(FALSE일시 제외됨)

		FROM [테이블명]
		INNER JOIN [테이블명]
		ON 조건식
		INNER JOIN [테이블명]
		ON 조건식
		…

2. 외부 조인(Outer Join)
   
		ON 절에 작성된 조건식이 FALSE 일지라도 모든 정보를 조회해야 할 때 사용한다.

		- LEFT OUTER JOIN
			선행 테이블의 모든 정보를 가져오고 싶을 때 사용한다.
		- RIGHT OUTER JOIN
			후행 테이블의 모든 정보를 가져오고 싶을 때 사용한다.

### 2-3 DCL
	DCL (Data Control Language): 데이터 제어 언어
 	데이터베이스에 대한 보안 목적으로 사용된다.

명령어 | 내용
--- | ---
GRANT | 권한을 정의할 때 사용
REVOKE | 권한을 제거할 때 사용

### 2-4 TCL
	TCL (Transaction Control Language): 트랜잭션 제어 언어

명령어 | 내용
--- | ---
COMMIT | 진행된 작업을 적용시킬 때 사용하는 명령어
ROLLBACK | 진행된 작업을 다시 처음으로 되돌릴 때 사용하는 명령어
SAVEPOINT | COMMIT 혹은 ROLLBACK을 특정 부분까지만 반영하기 위한 명령어
